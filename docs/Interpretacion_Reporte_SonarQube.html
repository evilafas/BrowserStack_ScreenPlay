¿Cómo habilitar la cobertura?

Añadir a build.gradle: plugin 'jacoco', configurar jacocoTestReport para generar XML, y agregar en sonarqube la propiedad "sonar.coverage.jacoco.xmlReportPaths". Luego ejecutar ./gradlew test jacocoTestReport sonar.

Glosario de Términos SonarQube

Quality Gate
Conjunto de condiciones que el proyecto debe cumplir para considerarse "apto". Si falla, el pipeline CI/CD puede bloquearse. Estado actual: PASSED.
Code Smell
Problema de mantenibilidad. El código funciona pero es difícil de leer, entender o modificar. No es un bug.
Bug
Código que casi seguramente producirá un comportamiento incorrecto en ejecución. Estado actual: 0.
Vulnerability
Fallo de seguridad que podría ser explotado por un atacante. Estado actual: 0.
Security Hotspot
Código sensible desde el punto de vista de seguridad que requiere revisión humana. Estado actual: 0.
Technical Debt (Deuda Técnica)
Tiempo estimado necesario para corregir todos los code smells. Actualmente: 137 minutos (0.5% del tiempo total del proyecto).
NCLOC
Non-Commented Lines of Code. Líneas de código reales excluyendo comentarios y líneas en blanco. Proyecto: 911.
Rating A–E
A = 0 bugs/vuln. B = 1 bug menor. C = bugs mayores. D = críticos. E = bloqueantes. El proyecto tiene A en todas las dimensiones.
Effort (Esfuerzo)
Tiempo estimado para corregir un issue específico, calculado automáticamente por SonarQube según el tipo de problema.
CRITICAL / MAJOR / MINOR / INFO
Severidad del issue. CRITICAL y MAJOR impactan la mantenibilidad del equipo. MINOR y INFO son mejoras de calidad opcionales.
